# 终端聊天室需求文档 v2.1（最终版）

## 1. 项目概述

**项目名称：** 终端聊天室 (Terminal Chatroom)

**核心定位：**
- 面向开发者的终端加密聊天室
- 单一"大厅"频道，所有在线用户实时交流
- 端到端加密，服务端仅负责转发和存储

**版本范围：** v1.0 基础版本

---

## 2. 核心功能

### 2.1 用户接入
- **认证方式：** 仅用户名
- **冲突处理：** 自动添加数字后缀
  ```
  用户 "alice" 登录 → alice
  第二个 "alice" 登录 → alice_1
  第三个 "alice" 登录 → alice_2
  ```
- **密钥管理：**
  - 首次登录时生成密钥对（公钥+私钥）
  - 公钥上传到服务器
  - 私钥保存在本地 `~/.chatroom/keys/username.key`

### 2.2 消息功能
- **聊天模式：**
  - ✅ 大厅频道（所有人可见）
  - ❌ 不支持私聊
  - ❌ 不支持多频道

- **消息格式：**
  - ✅ 纯文本
  - ❌ Markdown 支持（留待 v2.0）
  - ❌ 文件传输（留待后续版本）

- **离线消息：**
  - ❌ 不推送离线期间的消息
  - ✅ 消息永久保存在服务端（加密状态）
  - 用户可选择性查询历史消息（手动命令）

### 2.3 实时通信
- 在线用户发送的消息，所有其他在线用户实时接收
- 消息按服务器接收时间排序
- 终端滚动显示新消息

## 3. 技术实现方案

### 3.1 架构设计

```
┌─────────────┐          ┌──────────────┐          ┌─────────────┐
│  终端客户端  │ ←──WSS──→ │  聊天室服务器  │ ←──WSS──→ │  终端客户端  │
│   (alice)   │          │   (Golang)   │          │    (bob)    │
│             │          │              │          │             │
│  私钥(本地) │          │  公钥存储    │          │  私钥(本地) │
│  加密/解密  │          │  消息转发    │          │  加密/解密  │
└─────────────┘          │  消息存储    │          └─────────────┘
                         └──────────────┘
```

### 3.2 服务端技术栈

**语言：** Golang

**官方包使用：**
- `net/http` - HTTP/WebSocket 服务
- `crypto/rsa` 或 `crypto/ed25519` - 非对称加密
- `crypto/aes` - 对称加密（混合加密方案）
- `encoding/json` - 消息序列化
- `sync` - 并发控制
- `time` - 时间戳
- `os` / `io` - 文件存储

**核心模块：**
1. **连接管理器** - 维护 WebSocket 连接池
2. **用户管理器** - 用户注册、在线列表、公钥分发
3. **消息路由器** - 接收消息并广播给所有在线用户
4. **存储引擎** - 持久化加密消息到文件

### 3.3 客户端技术栈

**形态：** 单一可执行文件（Go 编译）

**支持平台：**
- Linux (amd64, arm64)
- macOS (amd64, arm64)
- Windows (amd64)

**核心功能：**
1. **密钥生成与管理** - 首次运行时生成密钥对
2. **WebSocket 客户端** - 连接服务器并保持长连接
3. **消息加密引擎** - 用所有在线用户的公钥加密消息
4. **消息解密引擎** - 用自己的私钥解密接收到的消息
5. **终端 UI** - 实时显示消息，底部输入区

**终端显示示例：**

## 4. 加密方案详解

### 4.1 混合加密架构

考虑到性能和安全性平衡，采用 **混合加密方案**：

```
发送消息流程：
1. 生成随机 AES-256 密钥（32字节）
2. 用 AES-GCM 加密消息内容
3. 用每个在线用户的公钥（Ed25519/RSA）加密 AES 密钥
4. 发送到服务器：
   {
     "from": "alice",
     "timestamp": 1234567890,
     "aes_encrypted_message": "base64...",  // AES 加密的消息
     "recipients": [
       {
         "to": "bob",
         "encrypted_aes_key": "base64..."    // Bob 公钥加密的 AES 密钥
       },
       {
         "to": "charlie",
         "encrypted_aes_key": "base64..."
       }
     ]
   }
```

**优势：**
- AES 加密速度快，适合加密长消息
- 只需加密一次消息内容
- 非对称加密只用于加密小的 AES 密钥（32字节）
- 100 用户场景下性能完全够用

### 4.2 密钥算法选择

**默认算法：Ed25519（推荐）**
- 公钥 32 字节，私钥 64 字节
- 签名/验证速度极快
- 安全性高（128位安全级别）
- Go 官方包 `crypto/ed25519` 支持

**可选算法：**
- RSA-2048/4096（`crypto/rsa`）
- ECDSA P-256（`crypto/ecdsa`）

**切换方式：**
```bash
# 客户端命令
/crypto ed25519    # 切换到 Ed25519
/crypto rsa2048    # 切换到 RSA-2048
```

### 4.3 安全特性

✅ **端到端加密** - 服务器无法解密消息内容
✅ **消息签名** - 防止消息伪造（发送者用私钥签名）
✅ **传输层加密** - WSS (WebSocket over TLS)
✅ **前向保密** - 每条消息使用独立的 AES 密钥
❌ **完美前向保密 (PFS)** - v1.0 不支持，后续考虑

## 5. 通信协议

### 5.1 传输协议
**WebSocket over TLS (wss://)**

### 5.2 消息格式（JSON）

#### 1. 用户注册/登录
```json
// 客户端 → 服务器
{
  "type": "register",
  "username": "alice",
  "public_key": "base64_encoded_public_key",
  "algorithm": "ed25519"
}

// 服务器 → 客户端
{
  "type": "register_response",
  "success": true,
  "assigned_username": "alice",  // 或 alice_1 如果冲突
  "online_users": ["bob", "charlie"]
}
```

#### 2. 获取公钥
```json
// 客户端 → 服务器
{
  "type": "get_pubkeys",
  "users": ["bob", "charlie"]  // 空数组表示获取所有在线用户
}

// 服务器 → 客户端
{
  "type": "pubkeys",
  "keys": {
    "bob": {
      "public_key": "base64...",
      "algorithm": "ed25519"
    },
    "charlie": {
      "public_key": "base64...",
      "algorithm": "ed25519"
    }
  }
}
```

#### 3. 发送消息
```json
// 客户端 → 服务器
{
  "type": "message",
  "from": "alice",
  "timestamp": 1709366625,
  "aes_encrypted_message": "base64_ciphertext",
  "recipients": [
    {
      "to": "bob",
      "encrypted_aes_key": "base64..."
    },
    {
      "to": "charlie",
      "encrypted_aes_key": "base64..."
    }
  ],
  "signature": "base64_signature"
}

// 服务器 → 所有在线用户（包括发送者自己）
{
  "type": "message",
  "from": "alice",
  "timestamp": 1709366625,
  "aes_encrypted_message": "base64_ciphertext",
  "encrypted_aes_key": "base64...",  // 针对接收者的密钥
  "signature": "base64_signature"
}
```

#### 4. 用户上线/下线通知
```json
{
  "type": "user_online",
  "username": "alice",
  "public_key": "base64...",
  "algorithm": "ed25519"
}

{
  "type": "user_offline",
  "username": "alice"
}
```

## 6. 数据存储

### 6.1 服务端存储

**存储内容：** 加密后的消息（服务器无法解密）

**存储格式：** JSON Lines (.jsonl)
```
{"timestamp":1709366625,"from":"alice","message":"base64...","recipients":[...]}
{"timestamp":1709366630,"from":"bob","message":"base64...","recipients":[...]}
```

**文件路径：** `./data/messages.jsonl`

**公钥存储：** 内存 + 文件备份
```json
// ./data/users.json
{
  "alice": {
    "public_key": "base64...",
    "algorithm": "ed25519",
    "registered_at": 1709366625
  },
  "bob": {
    "public_key": "base64...",
    "algorithm": "ed25519",
    "registered_at": 1709366630
  }
}
```

### 6.2 客户端存储

**密钥文件：** `~/.chatroom/keys/`
```
~/.chatroom/
├── keys/
│   ├── alice.key       # 私钥（权限 0600）
│   └── alice.pub       # 公钥
└── config.json         # 配置文件
```

**配置文件示例：**
```json
{
  "username": "alice",
  "server": "wss://chat.example.com:8443",
  "algorithm": "ed25519",
  "key_path": "/Users/alice/.chatroom/keys/alice.key"
}
```

## 8. 技术约束与边界

### 8.1 明确支持
✅ 100 并发用户以下
✅ 纯文本消息
✅ 大厅频道（单频道）
✅ 端到端加密
✅ 实时消息推送
✅ 用户名冲突自动处理
✅ 消息服务端存储
✅ 加密算法可切换

### 8.2 明确不支持（v1.0）
❌ 离线消息推送
❌ 私聊功能
❌ 多频道
❌ Markdown 格式
❌ 文件传输
❌ 消息编辑/撤回
❌ 用户权限管理
❌ 消息搜索

### 8.3 性能目标
- 并发用户：< 100
- 消息延迟：< 500ms（同机房）
- 内存占用：服务端 < 100MB，客户端 < 50MB
- 不考虑极端性能优化
